Opdrachtomschrijving Website Bouwen met HTML/CSS

In deze opdracht ga je in tweetallen of alleen een soort kruising maken tussen een digitaal klassenboek (denk aan Magister) en een sociaal netwerk zoals Facebook. Je kan op deze site dus niet alleen cijfers, roosters en huiswerk zien, maar ook persoonlijke profielpagina's van je medeleerlingen. Deze opdracht is verdeeld in twee tussenopdrachten waarin iedere leerling een eigen profielpagina maakt en een eindopdracht waarin deze pagina's worden samengevoegd tot een gehele site.

Deadlines:

  • 26-11-2025 (21:00): Tussenopdracht 1
  • 10-12-2025 (21:00): Tussenopdracht 2
  • 7-1-2026 (21:00): Eindopdracht

Cijferopbouw:

Je uiteindelijke cijfer voor dit PO wordt voor een derde bepaald door de tussenopdrachten en voor twee derde door de eindopdracht die je inlevert. De maximale punten die te behalen zijn, zijn als volgt verdeeld:

1 + 1,5 (tussenopdracht 1) + 1,5 (tussenopdracht 2) + 1,5 (eisen eindopdracht) + 4,5 (verdere beoordelingscriteria eindopdracht) = 10.

Tussenopdrachten:

Voor de tussenopdrachten geldt een pass/fail-systeem. Per tussenopdracht verdien je 1,5 punt voor je uiteindelijke PO-cijfer, als je deze op tijd inlevert en deze inhoudelijk in orde is (pass). Bij een fail krijg je hier geen punten voor. Je haalt een fail op je tussenopdracht in de volgende gevallen:

  • Je levert de tussenopdracht niet op tijd in.
  • Je ingeleverde tussenopdracht voldoet duidelijk niet aan de eisen.
  • Je ingeleverde tussenopdracht is duidelijk met ChatGPT of andere genAI gemaakt.

Als je alle tussenopdrachten succesvol inlevert heb je dus al minstens een 4 (1 + 1,5 + 1,5 = 4) voor je PO. Als je geen enkele tussenopdracht succesvol inlevert, haal je maximaal een 7 (10 - 1,5 - 1,5 = 7).

Eindopdracht:

Eisen (1,5 punt te behalen):

Zie de omschrijving van de eindopdracht.

Verdere beoordelingscriteria (4,5 punt te behalen):

  • Hoe uitgebreid is je site? Voldoe je enkel aan de basisvereisten of heb je zelf nog dingen toegevoegd?
  • HTML-/CSS-vaardigheid. (Pas je alleen toe wat in de les aan bod is gekomen of laat je zien dat je meer kan?)
  • Gebruikerservaring: Is de site mooi/overzichtelijk? Kan de gebruiker dingen aanpassen? Heeft alles een logische naam? Werken alle links?
  • Netheid: Staan alle bestanden op een logische plek/is je mapstructuur logisch? Gebruik je comments in je code? Is je code overzichtelijk?

Inleveren:

Alle opdrachten lever je in door de laatste versie van je project voor de desbetreffende inleverdeadline volledig op te slaan, te committen en naar github te pushen. Op GitHub kan ik zien wat je laatste push voor de inleverdeadline was en die zal ik beoordelen.

Te laat inleveren:

Wanneer je een deadline mist zonder van te voren goedgekeurde reden, gaat er 0,5 punt van je cijfer af per werkdag. Voor de tussenopdrachten betekent dit dat er na drie werkdagen geen punten meer voor te halen zijn. Voor de eindopdracht geef ik geen punten als deze vier of meer werkdagen te laat wordt ingeleverd.

Fraude en plagiaat:

Wanneer je code van een ander (in de klas of op het internet) overneemt telt dit als plagiaat. Dat is ook het geval wanneer je code inlevert die door ChatGPT, CoPilot of andere genAI gegenereert is. Bij vermoeden hierop krijg je geen punten voor het desbetreffende onderdeel tenzij je kan aantonen dat je zelf, zonder hulpmiddelen, over de HTML-/CSS-vaardigheid beschikt die je in je PO laat zien.
