# website-bouwen

## Welkom

Welkom bij de Github repository van jouw eigen websiteproject. Wat je nu leest, staat als standaardtekst alleen nog online bij Github, op de plek waar jullie je wijzigingen met elkaar kunnen delen en waar ik het uiteindelijk ook beoordeel.

## Clonen

De volgende stap is dat je deze repository kloont naar jouw eigen computer. Dit maakt een kopie van alle bestanden (tot nu toe alleen nog deze README en een paar plaatjes) en zorgt ervoor dat je wijzigingen kunt aanbrengen.

Je kunt dit als volgt doen in VS Code: klik op Source control in de linkerbalk, en vervolgens op `Clone repository`. Typ vervolgens in het balkje dat bovenaan het scherm verschijnt de URL of naam van de repository (bijvoorbeeld: `heemlanden/2324-infa3p-website-bouwen-docent`) en selecteer daarna `Clone from Github`. Deze naam kun je bovenaan deze pagina vinden. Mocht je de pagina weggeklikt hebben, dan kun je jouw repositories altijd terugvinden door in te loggen op Github en naar https://github.com/heemlanden te gaan.



![Clone screenshot](images/clone.png)

## Wijzigen

Tijd om je eerste wijzigingen aan te brengen! Maak een nieuw bestand aan en noem het `index.html`. Typ ! in het bestand en druk daarna op de `ENTER`-toets. Vervolgens voeg je in de body een tekst toe, bijvoorbeeld 'Dit wordt mijn website', en sla het bestand op met `Ctrl + S` (of `File` > `Save`).

![Add file screenshot](images/addfile.png)
![index.html file](images/webpage.png)

## Committen

Laten we jouw wijzigingen committen naar de repository op jouw eigen computer. Dit kun je ook weer in de Source control sectie doen aan de linkerkant. Als het goed is zie je een lijstje `Changes` met alle gewijzigde bestanden. Je neemt nu de volgende stappen:

* Klik op het plusje `+` naast de bestanden die je wilt committen (deze verschijnt als je er met de muis overheen beweegt). Het bestand verhuist dan van `Changes` naar  `Staged Changes`. Dit noem je 'staging'.
* Voeg vervolgens een bericht toe waarin je de wijzigingen beschrijft in het tekstveld boven de Commit knop. Deze tekst heet de 'commit message'.
* Klik tenslotte op de blauwe `Commit` knop.

![Commit screenshot](images/commit.png)

## Pushen

Nu rest (hopelijk! - bij geen wijzigingsconflicten) nog slechts een laatste stap om jouw code te publiceren naar de repository op de Github website!
Klik daarvoor op `Push`.

![Push screenshot](images/push.png)

 > [!NOTE] 
 > Als iemand anders al voor jou wijzigen gemaakt heeft, kan het zijn dat je deze eerst moet binnenhalen (met Pull) voordat je jouw wijzigingen weer kan pushen. Mochten jullie allebei in dezelfde bestanden iets gewijzigd hebben, dan moet je mogelijk een merge uitvoeren: dan moet je namelijk de wijzigingen met elkaar verenigen.
